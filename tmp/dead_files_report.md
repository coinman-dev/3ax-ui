## 2. Вероятно ненужные файлы (Probably Remove)

| Файл / Путь | Причина |
| :--- | :--- |
| `x-ui.rc` | Скрипт инициализации для устаревших систем (без systemd). Все современные дистрибутивы Linux используют `.service` файлы. Однако скрипт упоминается в `install.sh` как альтернатива. Если поддержка не-systemd систем не требуется — можно удалить. |
| `windows_files/SSL/Win64OpenSSL_Light-3_6_0.exe` | Установочный файл OpenSSL весом ~5MB. Он копируется в Windows-архив при релизе. Возможно, лучше предоставить инструкцию по установке актуальной версии OpenSSL вручную. |
| `README.ar_EG.md`, `README.es_ES.md`, `README.fa_IR.md`, `README.zh_CN.md` | Эти переводы содержат ссылки на старые ресурсы (логотипы 3x-ui), которые предлагаются к удалению. Если документация на этих языках не поддерживается — она вводит в заблуждение. |
| `CONTRIBUTING.md` | Стандартный файл, но часто остается пустым или содержит общие фразы. Если в проекте нет активного комьюнити контрибьюторов, может быть излишен. |

## 3. Нужные файлы для инфраструктуры (Keep)

Эти файлы необходимы для корректной работы автоматизированных процессов сборки образов и деплоя.

| Файл / Путь | Причина |
| :--- | :--- |
| `Dockerfile`, `docker-compose.yml`, `DockerInit.sh`, `DockerEntrypoint.sh` | **Полностью реализованная поддержка Docker**. Существует отдельный workflow (`.github/workflows/docker.yml`), который собирает и публикует образы для 5 архитектур в GHCR. Удаление этих файлов приведет к поломке CI/CD и невозможности использования проекта в контейнерах. |

---

## Резюме
Удаление файлов из раздела "Полностью ненужные" позволит сократить размер репозитория (особенно за счет изображений в `media/`) без какого-либо ущерба для функциональности или актуальной документации. Файлы Docker должны быть сохранены, так как они являются частью активной инфраструктуры проекта.
